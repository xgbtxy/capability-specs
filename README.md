# capability-specs

希望有一天，它能成为你日常的备用包。

当你遇到头疼的问题，打开它，它会给你思路，也会给你解决问题的能力。

这个仓库尽量写得精简、直接。

- 目录名字，代表开始的思路
- 里面的文件，代表解决的能力

如果你对一个问题不太了解，也没关系。把项目或问题丢给 AI，它会自己去找合适的目录和能力。

这里的每一个 spec，基本都代表“某个强工具可以解决的一类场景”。场景一定覆盖不完，所以这个项目适合大家一起慢慢补全。

这个仓库现在只保留两个核心原则：

- 目录名字要清晰
- 里面要有对应能力

其他形式，不设限。

如果你以后要继续维护这个仓库，可以先看 [templates/维护这个仓库的Prompt.md](./templates/维护这个仓库的Prompt.md)。
