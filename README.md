# capability-specs

一个文件夹一个能力：沉淀 Spec 索引与实战笔记，让 AI 快速落地真实任务。

## 这个仓库在做什么
互联网的信息很深、很散。这个项目把高价值方法整理成可复用 Spec，目标是**更快实现一件事**，而不是只停留在概念层。

## 核心思路
- 索引：按场景组织经典工具与资料入口
- 笔记：记录可执行步骤、关键参数、常见坑位
- 落地：让 AI 按 Spec 调用工具链，直接完成任务

## 目录约定
- `specs/<capability>/`：一个能力一个文件夹
- 每个能力目录建议包含：
  - `spec.yaml`（能力定义）
  - `README.md`（实战说明）
  - `examples/`（输入输出样例）
  - `notes.md`（迭代笔记，可选）

## 共建方式
如果你有独特想法或更优解，欢迎一起维护：提交 Issue / PR，把“能复用、能执行、能迭代”的方法沉淀下来。
