# environment-setup

## 目标

让执行者具备处理静态虚拟机镜像所需的最小命令行环境。

---

## 1. 必要软件

### 1.1 qemu-img

用途：

- 识别镜像基本信息
- 转换镜像格式

建议验证：

```powershell
qemu-img --version
qemu-img info <image>
```

---

### 1.2 libguestfs 工具组

推荐至少具备：

- `guestfish`
- `guestmount`

用途：

- 挂载镜像文件系统
- 提取指定文件
- 在离线状态下修改 Linux 系统文件

建议验证：

```powershell
guestfish --version
guestmount --version
```

---

### 1.3 可选辅助工具

- `qemu-nbd`
- `7z` 或其他归档工具

用途：

- 将镜像暴露为块设备
- 提取 `ova`、压缩包或其他容器里的真实磁盘文件

---

## 2. 访问前提

执行前建议确认：

- 已拿到镜像文件或虚拟机目录
- 有足够磁盘空间保存导出文件和转换后的镜像
- 当前用户具备访问镜像文件的权限

---

## 3. 当前建议支持范围

优先支持：

- `ova`
- `ovf`
- `vmdk`
- `qcow2`
- `raw`
- `vhdx`

---

## 4. 开始前检查

- [ ] 知道当前拿到的是单文件还是一整目录
- [ ] 知道本次目标是提文件、转格式，还是恢复访问
- [ ] 确认镜像文件可读
- [ ] 确认有输出目录可保存提取结果或转换结果
