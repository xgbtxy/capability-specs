# Linux离线恢复登录访问-SPEC

版本：v1.0  
状态：正式  
适用对象：忘记 Linux 虚机登录方式、但手里持有离线镜像的执行者

---

## 1. 问题定义

当执行者忘记 Linux 虚机的密码或可用登录路径时，如果手里已有离线镜像，很多时候不必进入单用户模式，也不必强依赖图形化工具。

但这里真正的问题不是“把旧密码解出来”，而是：

**如何在离线状态下恢复系统登录访问。**

---

## 2. 目标

1. 挂载 Linux 根文件系统
2. 确认目标用户和认证相关文件
3. 采用恢复访问而不是提取旧秘密的方式重新建立登录入口
4. 记录修改内容，便于后续回溯

---

## 3. 推荐恢复思路

优先考虑：

- 重置目标用户访问方式
- 注入新的 `authorized_keys`
- 修复明显错误的认证配置

不推荐把目标写成：

- 提取旧密码
- 解密旧秘密

---

## 4. 标准流程

### 第 1 步：挂载 Linux 根文件系统

先确认：

- 哪个分区是根分区
- 当前发行版和目录结构

### 第 2 步：定位认证相关文件

常见会涉及：

- `/etc/shadow`
- `/etc/passwd`
- `/root/.ssh/authorized_keys`
- `/home/<user>/.ssh/authorized_keys`
- SSH 或 PAM 相关配置

### 第 3 步：恢复访问

恢复方式应围绕“重新进入系统”，而不是“拿到旧秘密”。

### 第 4 步：记录与说明

必须记录：

- 修改了哪些文件
- 采用了哪种恢复方式
- 后续进入系统后还应做什么

---

## 5. 输出要求

最少应包含：

- 目标镜像
- 恢复方式
- 修改文件清单
- 最小验证说明

---

## 6. 安全边界

禁止：

- 把离线恢复写成旧凭证提取流程
- 不记录修改痕迹
- 未确认发行版和文件结构就盲改

允许：

- 在离线状态下重建登录入口
- 明确把当前版本标记为待持续实战打磨
