# PPT内容生成与迭代

把“根据输入材料生成一份可汇报的 PPT，或在现有 PPT 上继续迭代某一部分内容”的链路拆成可复用的小能力。

## 解决什么问题

- 根据文档、提纲、笔记生成整份 PPT
- 在已有 PPT 上补几页、改一章、调局部版式
- 输出真正可继续编辑的 `.pptx`

## 推荐工具

- Node.js：
  适合运行 PPT 生成脚本
  官方链接：[下载与安装](https://nodejs.org/en/download/package-manager) / [文档](https://nodejs.org/en/docs/)
- PptxGenJS：
  适合生成真正可编辑的 `.pptx`
  官方链接：[介绍](https://gitbrent.github.io/PptxGenJS/docs/introduction/) / [快速开始](https://gitbrent.github.io/PptxGenJS/docs/quick-start/)

## 最小使用方式

1. 先判断这次是整份新建还是局部迭代
2. 把输入材料整理成页面结构
3. 用 PptxGenJS 生成初稿或增量页面
4. 导出并检查最终 `.pptx` 是否可交付

## 为什么单独建这个文件夹

用户真实要的通常不是“学会某个 PPT 工具”，而是：

- 给一份输入，生成一份能直接汇报的 PPT
- 在现有 PPT 上补几页、改一章、优化某些页面
- 持续迭代，而不是每次都重做整份

所以这里不把能力写成“只会一键生成整份 PPT”，而是同时覆盖：

- 新建整份
- 局部生成
- 局部改写
- 持续迭代

## 适用对象

- 写材料后需要快速产出 PPT 的人
- 已有文档、纪要、方案，希望自动整理成汇报页的人
- 已有一版 PPT，希望追加页面或改写局部内容的人

## 建议使用顺序

1. 识别是新建整份还是局部迭代
2. 把输入材料整理成页面结构
3. 生成新页面或整份初稿
4. 更新已有PPT内容与版式
5. 导出并检查最终可交付版本

## 设计原则

- 从用户结果出发，而不是从工具名出发
- 同时支持“整份生成”和“局部迭代”
- 优先输出真正可继续编辑的 `.pptx`
- 页面结构、信息密度、视觉统一要比单页炫技更重要
