# tools-and-safety

## 目标

定义“PPT内容生成与迭代”中的推荐工具路径、安全边界和常见误区。

---

## 1. 工具优先级

推荐顺序：

1. `PptxGenJS`
2. 页面结构中间层，例如 Markdown / JSON 大纲
3. 辅助脚本

原因：

- `PptxGenJS` 最适合输出真正可继续编辑的 `.pptx`
- 中间层适合先整理每页讲什么，再落到正式版式
- 辅助脚本可以处理图片、图表、文本清洗，但不应喧宾夺主

---

## 2. 安全策略

### 2.1 任务识别策略

允许：

- 先判断是整份新建还是局部迭代
- 在已有 PPT 基础上只改指定部分

禁止：

- 不问范围就默认重做整份 PPT
- 把“补 3 页”理解成“整份重排”

### 2.2 页面密度策略

允许：

- 根据材料压缩到页面级表达
- 对一页内容做取舍和提炼

禁止：

- 机械把原文整段贴进 PPT
- 一页堆满文字导致无法汇报

### 2.3 可交付策略

允许：

- 输出真正可编辑的 `.pptx`
- 做导出后的基本检查

禁止：

- 用整页图片拼成伪 PPT 当正式交付件
- 不检查标题溢出、文本遮挡、图片变形

---

## 3. 常见失败点

### 3.1 只会生成整份，不会局部改

结果：

- 用户原本只想补一章，却被整份重写

### 3.2 结构没整理好就直接生成

结果：

- 页数失控
- 页面重复
- 章节顺序混乱

### 3.3 只管好看，不管可汇报

结果：

- 字太多
- 一页重点不明确
- 页面之间没有叙事节奏

---

## 4. 结论

这个场景最重要的不是“会不会生成 PPT”，而是始终知道：

- 这次是新建还是迭代
- 每一页到底要讲什么
- 最终输出是不是一个真的能拿去汇报的文件
