# 读取PR反馈并提炼行动项

解决“别人已经看完我的 PR，并给了评论或打回原因，但我看不懂到底下一步该做什么”。

## 文件说明

- [`spec.yaml`](./spec.yaml)
  定义这个能力的输入、输出和流程骨架
- [`读取PR反馈并提炼行动项-SPEC.md`](./读取PR反馈并提炼行动项-SPEC.md)
  展开说明如何读取 review、区分必改项和建议项

## 这个能力解决什么问题

对新手来说，review feedback 往往分不清：

- 哪些是必须改
- 哪些只是建议
- 哪些是风格意见
- 哪些其实是在问问题

这个能力的目标是：

**把 PR 中的评论、review 和打回原因整理成可执行的待办项。**

## 最小输出

- 必改项
- 建议项
- 待确认项
- 是否可以直接进入修订
