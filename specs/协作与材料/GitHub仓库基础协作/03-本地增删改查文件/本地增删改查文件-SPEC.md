# 本地增删改查文件-SPEC

版本：v1.0  
状态：正式  
适用对象：已经进入正确仓库与分支、准备在本地处理文件的新手执行者

---

## 1. 问题定义

进入正确分支之后，新手下一步通常就是在本地处理文件。但如果没有范围意识，很容易出现：

- 没找对文件
- 改了不该改的位置
- 新文件放错目录
- 删除文件时没意识到它还有引用关系

因此，需要一个本地文件操作能力，用于：

**在仓库上下文明确的前提下，安全完成文件的查看、新增、修改和删除。**

---

## 2. 目标

1. 明确本次任务涉及的文件范围
2. 在动手前先理解上下文
3. 清楚记录本次做了哪些增删改查
4. 确认是否可进入提交阶段

---

## 3. 标准流程

### 第 1 步：确认范围

先确认：

- 要处理哪个目录或文件
- 本次任务是否涉及新建文件
- 本次任务是否包含删除文件

### 第 2 步：先查再改

修改前应先阅读相关文件，避免：

- 不了解结构就直接改
- 在已有约定之外新增风格不一致内容

### 第 3 步：执行增删改

执行时应明确：

- 新增了什么
- 修改了什么
- 删除了什么

### 第 4 步：核对影响范围

检查：

- 是否动到了无关文件
- 是否可能影响引用关系
- 是否已经具备提交条件

---

## 4. 输出要求

最少应包含：

- 涉及文件列表
- 新增 / 修改 / 删除摘要
- 是否可进入提交阶段

---

## 5. 安全边界

禁止：

- 不看上下文就直接修改关键文件
- 删除文件后不说明影响
- 把“查看文件”与“修改文件”混成无法追踪的黑箱动作

允许：

- 先阅读再修改
- 在必要时指出当前任务范围仍不清晰
