# 根据反馈修订并更新PR

解决“别人已经指出问题，我也准备改了，但我不知道改完后怎么继续更新同一个 PR”。

## 文件说明

- [`spec.yaml`](./spec.yaml)
  定义这个能力的输入、输出和流程骨架
- [`根据反馈修订并更新PR-SPEC.md`](./根据反馈修订并更新PR-SPEC.md)
  展开说明如何在原分支继续修订、推送并回复 reviewer

## 这个能力解决什么问题

新手在这一步常见的困惑是：

- 不知道是不是要新开一个 PR
- 不知道改完后是不是还在原分支提交
- 不知道要不要回复评论
- 不知道怎么让 reviewer 看见已经处理了哪些问题

这个能力的目标是：

**在原工作分支上完成修订、更新同一个 PR，并清楚说明哪些反馈已经处理。**

## 最小输出

- 已完成的修订项
- 新提交说明
- PR 是否已更新
- 给 reviewer 的回复摘要
