# 认识仓库当前状态

解决“我刚进到一个 GitHub 仓库里，还不知道这个仓库现在是什么情况、我应该先看什么”。

## 文件说明

- [`spec.yaml`](./spec.yaml)
  定义这个能力的输入、输出和流程骨架
- [`认识仓库当前状态-SPEC.md`](./认识仓库当前状态-SPEC.md)
  展开说明仓库、分支、工作树、远端和最近变更的标准检查方式

## 这个能力解决什么问题

新手第一次进入一个仓库时，常见困惑包括：

- 不知道当前目录是不是正确仓库
- 不知道当前在哪个分支
- 不知道本地有没有未提交改动
- 不知道远端是谁
- 不知道最近做过什么事

这个能力的目标是：

**在开始动手前，把仓库当前状态看清楚。**

## 最小输出

- 仓库路径
- 当前分支
- 工作树是否干净
- 远端仓库信息
- 最近提交或近期变更摘要
