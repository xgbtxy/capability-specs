# 认识仓库当前状态-SPEC

版本：v1.0  
状态：正式  
适用对象：刚进入一个仓库、还不知道当前状态的新手执行者

---

## 1. 问题定义

进入一个仓库后，如果连当前状态都没搞清楚就直接动手，后面很容易出现：

- 在错误分支上工作
- 把别人的未提交改动看漏
- 不知道当前仓库连的是哪个远端
- 不知道最近一次变更是什么

因此，需要先做一次“仓库状态识别”，用于：

**在开始协作前明确仓库、分支、工作树和远端的基本事实。**

---

## 2. 目标

1. 确认当前目录是正确仓库
2. 明确当前分支和工作树状态
3. 明确远端仓库信息
4. 给出近期变更摘要

---

## 3. 标准流程

### 第 1 步：确认仓库身份

确认当前目录：

- 是否为 Git 仓库
- 仓库根目录在哪里

### 第 2 步：确认当前分支和工作树

确认：

- 当前分支
- 是否存在未提交改动
- 是否存在未跟踪文件

### 第 3 步：确认远端

确认：

- 是否配置了远端
- 默认远端是谁
- 指向哪个 GitHub 仓库

### 第 4 步：确认近期变更

至少要知道：

- 最近几次提交
- 当前分支是否明显落后或领先

### 第 5 步：输出结论

用自然语言告诉执行者：

- 现在在什么仓库
- 现在在哪个分支
- 当前是否适合继续下一步

---

## 4. 输出要求

最少应包含：

- 仓库路径
- 当前分支
- 工作树状态
- 远端仓库
- 最近提交摘要

---

## 5. 安全边界

禁止：

- 仓库状态没看清就直接开始编辑
- 把“有未提交改动”说成“可以放心切分支”

允许：

- 在发现异常状态后暂停，并先解释当前风险
