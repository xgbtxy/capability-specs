# 虚拟机镜像静态处理

把“已经拿到虚拟机镜像文件后，如何识别镜像、离线提取文件、转换格式，并在必要时恢复 Linux 登录访问”的链路拆成可复用的小能力。

## 解决什么问题

- 拿到镜像后先识别它到底是什么格式
- 从离线镜像中提取文件或目录
- 把镜像转换成更适合后续处理的格式
- 在 Linux 镜像上离线恢复登录访问

## 推荐工具

- QEMU / qemu-img：
  适合识别和转换镜像格式
  官方链接：[QEMU 文档](https://www.qemu.org/documentation/) / [qemu-img 文档](https://www.qemu.org/docs/master/tools/qemu-img.html)
- libguestfs：
  适合离线访问和修改虚拟机磁盘镜像
  官方链接：[官网](https://www.libguestfs.org/)
- guestfish：
  适合脚本化读取与修改镜像文件系统
  官方链接：[guestfish 文档](https://libguestfs.org/guestfish.1.html)
- guestmount：
  适合把镜像以只读方式挂到主机上导文件
  官方链接：[guestmount 文档](https://libguestfs.org/guestmount.1.html)

## 最小使用方式

1. 先识别镜像是容器、描述文件还是实际磁盘
2. 决定是直接提文件还是先转换格式
3. 用只读方式挂载并提取所需内容
4. 必要时再进入 Linux 离线恢复访问

## 为什么单独建这个文件夹

这组能力的目标不是讨论虚拟化概念，而是让 AI 能在命令行里直接处理静态镜像文件。

这里特别收敛到“静态处理”：

- 手里已经有 `ova`、`ovf`、`vmdk`、`qcow2`、`vhdx`、`raw` 等文件
- 重点是离线操作
- 不依赖图形化工具
- 不把能力写成凭证提取或秘密获取

## 适用对象

- 手里已经拿到虚拟机镜像文件的人
- 需要从镜像中提文件、看配置、导出文件的人
- 需要转换镜像格式以便后续挂载或启动验证的人
- 忘记 Linux 虚机登录方式、需要离线恢复访问的人

## 包含的能力

- [`01-识别镜像类型并确认可操作路径`](./01-识别镜像类型并确认可操作路径)
  解决“我手里的到底是什么文件，接下来该怎么处理”

## 建议使用顺序

1. 识别镜像类型并确认可操作路径
2. 离线从虚拟机镜像提取文件
3. 转换虚拟机镜像格式
4. Linux 离线恢复登录访问

## 设计原则

- 优先使用 AI 可操作的命令行工具
- 只处理静态镜像，不依赖图形化界面
- 重点解决“文件、格式、访问恢复”问题，不扩展到凭证提取
- 已做过的场景明确标注为已验证，没跑过的不伪装成已跑通
