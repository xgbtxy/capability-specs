# 转换虚拟机镜像格式-SPEC

版本：v1.0  
状态：正式  
适用对象：需要将现有镜像转换为更适合挂载、提取或启动验证格式的执行者

---

## 1. 问题定义

很多时候镜像本身没坏，但当前格式不方便下一步处理。例如：

- 工具链更适合 `qcow2`
- 当前拿到的是 `vmdk` 或 `vhdx`
- 需要把 `qcow2` 转成 `raw` 做其他处理

因此，需要一个稳定的格式转换能力，用于：

**把真实磁盘镜像转换为后续更好用的目标格式。**

---

## 2. 目标

1. 正确识别源格式
2. 正确选择目标格式
3. 生成新的镜像文件
4. 做最小可读性验证

---

## 3. 标准流程

### 第 1 步：识别源格式

优先使用：

```text
qemu-img info
```

### 第 2 步：执行转换

典型方式：

```text
qemu-img convert -O <target-format> <source> <output>
```

### 第 3 步：验证结果

至少要确认：

- 输出文件存在
- `qemu-img info` 能正常读取
- 目标格式与预期一致

---

## 4. 输出要求

最少应包含：

- 源格式
- 目标格式
- 输出路径
- 基本验证结果

---

## 5. 安全边界

禁止：

- 没确认真实磁盘文件就盲转
- 只说“命令执行成功”但不验证输出镜像

允许：

- 在转换前先复制或保留原始镜像
