# 完成一次PR协作闭环

解决“我已经完成了查看仓库、改文件、提 PR、收反馈、继续修订这一轮，现在怎么确认这次协作是不是走完整了”。

## 文件说明

- [`spec.yaml`](./spec.yaml)
  定义这个能力的输入、输出和流程骨架
- [`完成一次PR协作闭环-SPEC.md`](./完成一次PR协作闭环-SPEC.md)
  展开说明如何检查这一轮协作是否已经完成或仍卡在某一步

## 这个能力解决什么问题

很多时候执行者做了很多动作，但自己并不确定：

- 是不是已经完成这一轮该做的事
- PR 现在是在等 reviewer 还是还没推送完
- 是否还有没处理的反馈
- 现在到底该停下来等，还是继续改

这个能力的目标是：

**把一次 PR 协作从起点到当前状态串起来，给出“现在处于哪一步、下一步是什么”的闭环结论。**

## 最小输出

- 当前所处阶段
- 已完成步骤
- 未完成步骤
- 下一步建议
