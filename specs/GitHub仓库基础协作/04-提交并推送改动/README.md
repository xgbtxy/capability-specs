# 提交并推送改动

解决“我本地已经改完了一部分内容，现在怎么安全地提交并上传到远端分支”。

## 文件说明

- [`spec.yaml`](./spec.yaml)
  定义这个能力的输入、输出和流程骨架
- [`提交并推送改动-SPEC.md`](./提交并推送改动-SPEC.md)
  展开说明如何筛选文件、写提交信息和执行推送

## 这个能力解决什么问题

新手在这一步常见的担心是：

- 不知道哪些文件应该提交
- 不知道提交说明怎么写
- 不知道 `push` 到哪里
- 怕把无关文件一起传上去

这个能力的目标是：

**把本地真实改动整理成可解释的一次提交，并推送到正确远端分支。**

## 最小输出

- 本次将提交哪些文件
- 提交说明
- 推送目标分支
- 推送是否成功
