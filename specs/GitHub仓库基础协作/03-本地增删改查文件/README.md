# 本地增删改查文件

解决“我已经进到正确仓库和分支了，在本地到底怎么安全地看文件、加文件、删文件、改文件”。

## 文件说明

- [`spec.yaml`](./spec.yaml)
  定义这个能力的输入、输出和流程骨架
- [`本地增删改查文件-SPEC.md`](./本地增删改查文件-SPEC.md)
  展开说明查看、创建、修改、删除文件时的范围控制和安全边界

## 这个能力解决什么问题

很多新手不是不会写内容，而是不确定：

- 该去哪里看文件
- 新文件应该放哪
- 改文件前要不要先确认上下文
- 删除文件会不会删错

这个能力的目标是：

**在当前仓库和分支中，安全地完成本地文件的增删改查。**

## 最小输出

- 本次查看或修改的目标文件
- 新增、修改、删除动作摘要
- 是否影响其他相关文件
- 下一步是否可以进入提交
