# 读取PR反馈并提炼行动项-SPEC

版本：v1.0  
状态：正式  
适用对象：PR 已收到 review、但执行者还看不懂哪些必须改的人

---

## 1. 问题定义

PR review 阶段的反馈通常分散在多个位置：

- 顶层评论
- review 提交意见
- 行内评论
- requested changes

对新手来说，最难的不是看到评论，而是把这些评论整理成清晰行动项。

因此，需要一个专门的反馈阅读能力，用于：

**把分散的 reviewer 反馈整理成“必改 / 建议 / 待确认”的执行清单。**

---

## 2. 目标

1. 收集 PR 中的主要反馈
2. 区分强制修改和可选优化
3. 合并重复评论
4. 给出能直接进入修订的待办列表

---

## 3. 标准流程

### 第 1 步：读取 PR 元数据和评论

至少应读取：

- PR 当前状态
- 顶层评论
- review 结论
- 行内评论

### 第 2 步：分类反馈

建议至少分为：

- 必改项
- 建议项
- 待确认项

### 第 3 步：合并重复内容

若多个 reviewer 指向的是同一问题，应合并成一个行动项，避免执行者重复处理。

### 第 4 步：输出面向执行者的说明

输出应尽量避免 review 术语堆砌，而是直接说：

- 要改什么
- 为什么要改
- 哪些还需要确认

---

## 4. 输出要求

最少应包含：

- 必改项列表
- 建议项列表
- 待确认项列表
- 是否可以直接进入修订

---

## 5. 安全边界

禁止：

- 把 reviewer 的问题误当成已确认结论
- 漏掉 requested changes 中的明确动作
- 只复制原评论，不做提炼

允许：

- 将多个评论整合成执行语言
- 在不失真的前提下压缩冗余表述
