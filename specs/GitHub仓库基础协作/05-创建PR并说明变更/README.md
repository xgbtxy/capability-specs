# 创建PR并说明变更

解决“改动已经上传了，我怎么正式发起一个让别人审核的 PR”。

## 文件说明

- [`spec.yaml`](./spec.yaml)
  定义这个能力的输入、输出和流程骨架
- [`创建PR并说明变更-SPEC.md`](./创建PR并说明变更-SPEC.md)
  展开说明 base / head 关系、标题写法和正文结构

## 这个能力解决什么问题

对于新手来说，PR 最大的难点通常不是按钮在哪里，而是：

- 不知道 base 分支和 head 分支怎么对应
- 不知道 PR 标题怎么写
- 不知道正文里该写什么
- 不知道应该突出“我改了什么”和“别人该怎么看”

这个能力的目标是：

**基于已经推送好的远端分支，创建一个信息完整、可读性好的 PR。**

## 最小输出

- base 分支
- head 分支
- PR 标题
- PR 正文摘要
- PR 链接
