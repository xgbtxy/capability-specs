# 接手或切换分支

解决“别人已经给我安排好了分支，或者我要切到另一个分支，我要怎么把本地环境切到正确状态并开始工作”。

## 文件说明

- [`spec.yaml`](./spec.yaml)
  定义这个能力的输入、输出和流程骨架
- [`接手或切换分支-SPEC.md`](./接手或切换分支-SPEC.md)
  展开说明标准执行步骤、判断条件和风险边界

## 这个能力解决什么问题

新手最常见的卡点不是写代码，而是：

- 不知道当前自己在哪个分支
- 不知道别人说的分支名该怎么接
- 不知道本地是不是最新
- 怕一通操作把自己的改动弄丢

这个能力的目标是：

**把当前仓库切到正确分支，并确认可以安全开始修改。**

## 最小输出

- 当前仓库路径
- 当前所在分支
- 目标工作分支
- 是否已经同步到远端最新状态
- 是否可以开始修改
