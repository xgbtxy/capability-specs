# tools-and-safety

## 目标

定义“做事前后检查”中的推荐工具路径、安全边界和常见误区。

---

## 1. 工具优先级

推荐顺序：

1. 当前场景已经存在的原生命令行工具
2. 只读检查命令
3. 最小写入或最小回退操作

原因：

- 这组能力的重点是保障，不是再引入更多复杂工具
- 优先先看清状态，再决定是否动手

---

## 2. 安全策略

### 2.1 开始前先看清

允许：

- 先检查环境、输入和前置条件
- 在不满足条件时暂停开始

禁止：

- 没检查就直接进入主流程

### 2.2 范围先确定

允许：

- 先判断是整份、局部、单文件还是多对象

禁止：

- 不清楚范围就直接扩大处理

### 2.3 结果必须验证

允许：

- 对最终产物做最小可用性检查

禁止：

- 把“命令没报错”当成“结果能用”

### 2.4 失败必须留退路

允许：

- 明确写出排查顺序和回退方式

禁止：

- 失败后只说“重新试试”

---

## 3. 常见失败点

### 3.1 环境其实不满足

例如：

- 工具没装
- 路径写错
- 权限不够

### 3.2 一开始就做大了

例如：

- 用户只让补 3 页，却重做整份
- 本来只想提一个文件，却导出整盘

### 3.3 结果其实不能用

例如：

- 文件生成了但打不开
- 转换成功但无法挂载
- 导出了 PPT 但版式已经坏掉

---

## 4. 结论

这组能力最重要的不是“多做事”，而是：

- 开始前看清
- 动手前圈定
- 做完后验证
- 失败时能退
