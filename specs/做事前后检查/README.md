# 做事前后检查

把“开始前先看能不能做、做之前先判断范围、做完检查结果能不能用、失败时知道怎么排查和回退”拆成可复用的小能力。

## 解决什么问题

- 开始前不知道环境和前提是不是满足
- 动手前没判断范围，结果越做越大
- 做完后不知道结果到底能不能用
- 失败后只会重试，不知道怎么查和怎么退

## 推荐工具

- PowerShell：
  适合做环境检查、路径检查、结果验证和失败排查
  官方链接：[文档](https://learn.microsoft.com/powershell) / [Windows 安装](https://learn.microsoft.com/en-us/powershell/scripting/install/installing-powershell-on-windows?view=powershell-7.5) / [Linux 安装](https://learn.microsoft.com/en-us/powershell/scripting/install/installing-powershell-on-linux?view=powershell-7.5)
- 当前场景原生命令行工具：
  适合结合具体任务做最小检查和最小验证
  官方链接：复用对应能力里已经选好的官方工具链接

## 最小使用方式

1. 开始前检查环境、输入和输出路径
2. 判断这次到底是整份还是局部、单对象还是多对象
3. 做完后验证结果是否真的可用
4. 失败时按顺序排查，并保留最小回退路径

## 为什么单独建这个文件夹

很多能力失败，不是因为主流程不会做，而是因为：

- 一开始没检查环境
- 一开始没判断任务范围
- 做完后没验证结果
- 出问题后没有排查和回退路径

所以这组能力不是某个业务领域专用能力，而是所有场景都能复用的通用保障层。

## 适用对象

- 需要把能力从“会做”提升到“稳定可落地”的人
- 在不同场景里反复遇到环境问题、范围误判、结果不可用的人
- 想给其他能力补一层通用保障的人

## 包含的能力

- [`02-判断任务范围与处理模式`](./02-判断任务范围与处理模式)
  解决“这次到底要做多大范围、按哪种模式处理”
- [`03-做完检查结果是否可用`](./03-做完检查结果是否可用)

- [`04-失败时排查问题与回退`](./04-失败时排查问题与回退)

## 建议使用顺序

1. 开始前检查环境与前置条件
2. 判断任务范围与处理模式
3. 做完检查结果是否可用
4. 失败时排查问题与回退

## 设计原则

- 尽量做成跨领域通用能力
- 不把“检查”写成可有可无的一句提醒
- 明确区分“命令跑了”和“结果可用”
- 明确区分“临时失败”和“需要回退”
